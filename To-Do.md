# Resume Studio To-Do

Ideas worth keeping from the Reactive Resume review, intentionally deferred until the core local workflow is tighter.

## Layout and Organization

- Add item-level "Move to..." actions for guided-mode experience, projects, skills, and custom entries.
- Support splitting longer resumes into page-aware sections such as "Earlier Experience" or "Additional Projects."
- Add a visual page layout manager for non-ATS templates with page 1/page 2, main/sidebar placement, full-width pages, and sidebar width.
- Add guided custom sections for Board / Advisory, Speaking, Publications, Patents, Open Source, Selected Clients, and Earlier Experience.

## Design Power Tools

- Add an advanced scoped Custom CSS panel in Design.
- Provide safe starter selectors and snippets for compact headings, dividers, denser skills, and link styling.
- Warn in Review when custom CSS may reduce ATS readability or export safety.

## Sharing and Collaboration

- Add optional public read-only resume links after backend/storage architecture is ready.
- Add optional password protection for public links.
- Add basic private view/download statistics for shared links.
- Add one-click disable for any shared link.

## Optional AI

- Add local AI provider settings with user-supplied API key, provider, model, and base URL.
- Keep deterministic local review as the default.
- Show AI output as Markdown diffs and require explicit user approval before applying.
- Add strict prompts that never invent employers, dates, credentials, or metrics.

## Interoperability

- Add a visible schema documentation page for Resume Studio JSON.
- Add migrations for future Resume Studio JSON schema versions.
- Add import validation details so users can see what fields were accepted, ignored, or repaired.
