# Public Roadmap

This roadmap is not a promise. Priorities may change during public beta based on user feedback, security needs, maintenance cost, and product direction.

## Now

- Improve mobile editor layout and small-screen guardrails.
- Add and maintain E2E smoke coverage for launch-critical flows.
- Improve import limitation messaging and parser tests.
- Add browser-specific Print / Save as PDF tips.
- Improve accessibility checks for dialogs, focus behavior, and keyboard navigation.

## Next

- More robust import cleanup for DOCX and selectable-text PDFs.
- Stronger DOCX export fidelity with selected resume templates.
- Better Print / Save as PDF checks for margins, page size, and page breaks.
- Improved empty states and dashboard guidance.
- Application tracking improvements that connect job targets, versions, exports, and follow-ups.

## Later

- Better resume templates without adding sample resumes as part of this launch pass.
- More complete backup/restore E2E coverage.
- Browser compatibility notes for Firefox, Safari, Edge, and Chrome.
- Optional encrypted local backup with WebCrypto.
- Optional private sharing only after backend architecture, security controls, and privacy copy are ready.

## Ideas Under Consideration

- Optional AI assistance only if the privacy model is clear, disclosed, and user-controlled.
- Markdown syntax help for Resume Studio-specific directives.
- Page-aware section planning for longer resumes.
- More import validation details for accepted, ignored, repaired, or unparsed fields.
- Contributor-friendly test fixtures that use minimal fictional data only.

## Source Notes

The root `To-Do.md` remains a broader internal working list. This roadmap is the public-facing summary for contributors and public beta users.
