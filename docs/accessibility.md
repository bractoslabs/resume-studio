# Accessibility

Resume Studio should be usable with a keyboard, readable at small screen sizes, and understandable without relying only on color or icons.

The current public beta includes:

- Visible focus states
- Text labels or accessible labels for icon-only controls
- Dialog semantics for modals
- Escape-key handling for primary feedback/import dialogs
- Plainspoken warnings that do not rely only on color
- Mobile notes where full editing is less comfortable

## Reporting Accessibility Issues

Use the GitHub accessibility issue template for public, non-sensitive reports:

https://github.com/bractoslabs/resume-studio/issues/new?template=accessibility_issue.yml

For private context, email labs@bractos.com.

Do not include private resume content, personal contact information, job application details, or sensitive data in public issues, screenshots, recordings, or fixtures.

## Public Beta Note

Accessibility work is ongoing during the public beta. Focused reports with browser, assistive technology, and steps to reproduce are especially helpful.
