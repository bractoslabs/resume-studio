# GitHub Discussions Guidance

GitHub Discussions may be useful after public beta launch if community feedback grows. Discussions cannot be fully enabled from repo code, so maintainers need to turn it on in GitHub settings when ready.

## Suggested Categories

- Announcements
- Ideas
- Q&A
- Show and tell
- Help

## Issues vs. Discussions

Use GitHub Issues for actionable bugs, feature requests, accessibility problems, import/export issues, and documentation improvements.

Use Discussions for broader questions, early ideas, community feedback, and non-actionable help topics.

## Privacy Reminder

Do not post private resume content, personal contact information, job application details, or sensitive data in public Discussions.

For private feedback, email labs@bractos.com.
